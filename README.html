MBR Wiper & System Distorter

Analisi tecnica e comportamentale di un payload distruttivo Win32 in C++

⚠️ ATTENZIONE: Codice ad Alto Rischio

I sorgenti associati a questa pagina sono strutturati per sovrascrivere settori hardware critici ed eliminare file di sistema. Non eseguire l'applicazione su sistemi di produzione o macchine fisiche ospitanti. Utilizzare esclusivamente laboratori di malware analysis isolati.

Vettori di Distruzione

  • PhysicalDrive0: Accesso diretto via API al settore 0 (MBR) per invalidare la tabella delle partizioni.
  • Wiping dei Driver: Svuotamento mirato della directory System32\drivers per indurre il Blue Screen (BSOD).
  • Epurazione Font: Rimozione dei file vettoriali in C:\Windows\Fonts per compromettere il rendering dell'interfaccia.

Payload Visivo & Sonoro

  • GDI Meltdown: Loop infinito di alterazione dello schermo tramite combinazioni asincrone di BitBlt e PatBlt.
  • Anti-Chiusura: Creazione della finestra in modalità WS_EX_TOPMOST per inibire il normale utilizzo del desktop.
  • Audio Distorsivo: Riproduzione asincrona ciclica di risorse audio incorporate nel binario.

Meccanismi di Mitigazione nei Sistemi Moderni

Nelle moderne infrastrutture di sicurezza, l'impatto di questo codice viene intercettato o mitigato da diversi livelli del sistema operativo:

  • Secure Boot & GPT: I sistemi basati su partizionamento GPT non dipendono dall'MBR classico per l'avvio, impedendo il blocco totale del firmware tramite il solo settore 0.
  • TrustedInstaller ACLs: Le cartelle di sistema sensibili negano i permessi di scrittura e cancellazione anche all'utente Amministratore standard, richiedendo un esplicito ed elevato bypass dei privilegi per rendere efficace il wiping.
  • UAC & EDR: L'apertura di un handle diretto verso un disco fisico genera un alert immediato da parte delle soluzioni antivirus o dei sistemi di rilevamento euristico.
